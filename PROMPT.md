# AI 日报 · 报纸生成规范（PROMPT）

> 这份文件是给**任何模型**看的（Claude / GPT / Gemini / Qwen / DeepSeek 均可）。
> 只要把「今天的新闻素材」+ 本文件 + `profile.md` + `example-*.html` 一起给它，
> 它就能生成一份风格一致的报纸。**规则明确、块型有限、模板固定——版式的整齐靠这份规范，不靠模型即兴。**

---

## 0. 任务

把今天挑好的 5–8 条 AI 新闻，排成一份**印刷报纸风格的 HTML 单页**。
输出**一个完整的、自包含的 .html 文件**（CSS 内联、图片 base64 内嵌，手机也不裂）。

## 1. 输入

1. **今天的新闻素材**：每条含标题、正文事实、来源链接、（若有）原文配图 URL。
2. **`profile.md`**：读者是谁（身份/领域/目标/盲区/关注镜头）。**"对你"那一句要读它来写。**
3. **`example-2026-07-09.html`**：标准样板。**CSS 原样照抄，HTML 结构照它套。**

## 2. 选题与头条

- 选题两个镜头，各占一半（见 `profile.md` 的"镜头 A / B"）。过滤纯融资流水账。
- 只报**近 48 小时**（最多放宽到一周）的硬新闻。宁缺毋滥，不凑数、不报旧闻。
- **头条（只一条）** = 时效 × 影响面 × 可反推性 三项最高的那条。多条都硬时，选"读完最可能让读者改一个决策"的。

## 3. 写作规范（通讯社简明体）

每条固定三段：**正文 → 洞察 → 对你**。

- **正文**：1–2 句客观讲清发生了什么。读者可能不熟的术语，用括号顺手科普一句（不必每句都加）。
- **洞察（行业层，对所有读者一样）**：跳出这条新闻，讲它在行业里揭示了什么变化/机制/趋势。2–4 句。给判断，不复述。
  - 句式：*这件事说明〔行业变化〕，因为〔机制〕，所以接下来〔趋势判断〕。*
- **对你（读 `profile.md`，因人而变）**：承接洞察，落到"对这个读者具体意味着什么 / 能反推什么机会或动作"。1–2 句，必须具体到他的领域。
  - 若这条与读者无关：诚实写"这条与你当前无直接关系，但值得懂：〔为什么〕"（成长镜头，别硬套）。
- **禁止**：论文腔、名词化黑话（"结构化决策支持"）、空话（"值得关注""影响深远"）、卖萌金句、"也许/可能"开头下结论。

## 4. 图片规范

- **一条新闻一张原图**：原文有可用配图就必须扒下来用。图片**下载后 base64 内嵌**，不挂外链（防盗链/裂图）。
- **诚实四态**（读者永远知道图的来路）：
  1. 有贴题原图 → 用。图注：`图：〔内容〕　来源：〔媒体〕`
  2. 无可用配图 → 不放图，「来源」后加 `（本条无配图）`
  3. 有原图但不贴题（通用/资料/logo）→ **仍然用，但缩小、放次要位置**，图注写明 `图：原文配图，与本条内容无直接关联。来源：〔媒体〕`
  4. 自制示意图（仅原图缺失时才做）→ 图注写明 `示意图（本报制作），非新闻现场图。`
- **三档大小 = 重要度 × 想看清价值**：有人脸/表情/细节、读者想凑近看的 → 放大、靠视觉中心；通用图 → 缩小、放边角。分主图 / 中图 / 小图。
- **对角线错位**：同一模块相邻两条，图位一上一下（A 图在上、B 图在底），让两图走左上→右下的对角线，避免呆板。

## 5. 版式规范（报纸感，非网页流）

- **模块化互锁，禁止从上到下一条条堆。**
- **板块**：用 `### 板块名`（无 emoji），只留当天有内容的。如 `大模型 · Agent`、`出海 · 中日跨境`、`日本 · 企业 AI 落地`、`具身智能`、`拓展 · 利于成长`。板块名用大号无衬线 + 粗横线线眉（见样板）。
- **板块内 ≥2 条 → dogleg 互锁**：一条占"顶部通栏标题 + 左下正文"（L 形），另一条嵌右下角，中间发丝竖线分隔。单条 → 老实占一宽条（正文双栏排）。
- **有限块型**：hero（头条大块）/ 宽条 / 窄高 / 小块。按重要度给每条分块型，别每条都独一份怪形状（守一致性，别让人看着累）。
- **响应式**：宽屏多列、平板 2 列、手机 1 列，自动重排（CSS 已写好，照抄样板）。
- **眼动线铁律**：标题永远在最上（图不许压在标题之上）；来源链接一律 `target="_blank"` 新窗口打开。

## 6. 报头（配置）

- **报名**：读 `profile.md` 的 `报名` 字段（现占位"伟大航路 · AI 日报"，读者可随时改）。
- **报眉**：期号 + 日期 + 星期 + 一行栏目词。

## 7. 输出

- 一个完整 `.html`：`<style>` 原样抄 `example-2026-07-09.html` 的，`<body>` 结构照它套，只换当天内容与图片。
- 文件名 `YYYY-MM-DD.html`。
- **拿不准就对着 `example-2026-07-09.html` 比**——它是这套规范的标准答案。
