# 专属日报 · 报纸生成规范（PROMPT · 万能版）

> 给**任何模型**看（Claude / GPT / Gemini / DeepSeek / Qwen / Kimi / 豆包 均可）。
> 把「今天的新闻素材」+ 本文件 + `profile.md` + 对应样板 `example-*.html` 一起交给它，它就产出一份风格一致的报纸。
> **一份规范，换 `profile.md` 就换受众**：专业读者得到"AI 行业日报"，退休长辈得到"本地生活小报"，靠的是同一套版式和铁律。
>
> **选模型的建议**：国内 / 本地 / 中文新闻，优先用**国内大模型 + 联网**（DeepSeek / Qwen / Kimi / 豆包），更容易触达中文本地源；全球 / 英文新闻用 Claude / GPT / Gemini 皆可。**但换模型只在边际上有用——超本地八卦、当天真图这类"源头封闭"的内容，换哪个模型都难自动拿到（见 §5、§9），要坦诚标注、逐步啃，绝不造假。**

---

## 1. 任务
把今天挑好的 5–9 条新闻，排成一份**印刷报纸风格的 HTML 单页**，输出一个**自包含 .html**（CSS 内联、图片 base64 内嵌，手机不裂）。

## 2. 输入
1. **今天的新闻素材**：每条含标题、事实、来源链接、（若有）原文配图 URL。
2. **`profile.md`**：读者是谁 + 选题镜头 + 报名 + **版式档案**（决定腔调、板块、字号）。
3. **对应样板 `example-*.html`**：CSS 原样照抄，HTML 结构照它套。

## 3. 选题与头条
- 按 `profile.md` 的选题镜头选，各镜头大致均衡；过滤流水账。
- 只报**近 48 小时**（最多一周）的新鲜事。宁缺毋滥，不凑数、不报旧闻。
- **头条只一条** = 时效 × 影响面 × 与读者相关度 最高的。生活版里，**天气 / 安全 / 钱袋子**这类通常最该上头条。

## 4. 写作规范（腔调随 profile 变）
每条固定三段：**正文 → 看点 → 落到读者**。三段的"腔调"和"落点"由 profile 的版式档案决定：

| 版式档案 | 腔调 | 第三段叫 | 落到哪 |
|---|---|---|---|
| 专业版（如 AI 行业） | 通讯社简明体：短句、客观、去黑话 | **对你** | 读者的领域里能反推的机会 / 动作 |
| 生活版（如中老年本地） | 晚报暖体：亲切、口语、有画面 | **提醒您** | 读者能用上的贴心提示 / 该注意啥 / 怎么占便宜 |

**跨版本铁律：**
- **具体 > 抽象，是硬要求。** 多用数字、情节、画面。"暑托点 91 个"是坏例；"500 株粉荷 10 分钟被抢光"是好例。抽象的官方表述，改写成有人、有数、有画面的说法。
- **八卦 / 社会 / 防骗 / 奇闻**对某些读者（尤其中老年）是**核心板块，不是边角**。要有情节、有冲突、有"提醒您"的实用落点——既好看又有用。
- **禁止**：论文腔、名词化黑话（"结构化决策支持"）、空话（"值得关注""影响深远"）、卖萌金句、"也许 / 可能"开头下结论。

## 5. 图片规范
- **每期至少 2 张图**：5–9 条新闻里，总能挑出 2 条以上值得配图、也配得到好图的——头条和最有画面感的那条优先。原文没图就主动去找合适的图，但**烂图不许凑数**（见下），宁可某条不配也不许配一张让人摸不着头脑的。
- **什么叫"好图"（三个硬门槛，缺一不用）**：① **高清**——原始宽度 ≥800px，模糊、有水印、截图感重的一律不用；② **贴题**——图的主体必须就是新闻的主体（说某芯片政策就配该芯片/该公司 CEO，不许拿相邻型号的拆解图糊弄；说某公司就配该公司产品或 CEO，不许拿"泛 AI 概念图"）；③ **有编辑感**——学《经济学人》的配图直觉：**新闻有关键人物就优先配人物特写**（大公司 CEO 的高清人物照 Wikimedia 都有），产品新闻配产品官图，政策/数据类新闻找不到具象图就**不配**，抽象配图是减分项。
- **哪些新闻该配图（自己判断，别平均主义）**：人物驱动 / 产品发布 / 有现场（发布会、门店、机器人）→ 该配；纯政策、纯数据、纯观点 → 默认不配。头条尽量配。
- **一条新闻一张原图**：原文有可用配图就必须扒下来，**下载后 base64 内嵌**，不挂外链。
- **诚实四态**（读者永远知道图的来路）：① 贴题原图→用，图注 `图：〔内容〕　来源：〔媒体〕`；② 无图→不放，「来源」后加 `（本条无配图）`；③ 有图但不贴题→**直接不用**（不贴题=不上版）；④ 自制图→标 `示意图（本报制作）`。
- **本地 / 中文源的现实**：很多中文本地站**不给 og:image、正文靠 JS 加载**，原图常抓不到。抓不到时按序退：**用相关的开放版权资料图**（维基百科 / Wikimedia，图注标 `资料图`）→ 实在没有就不放图并标注。**绝不拿无关图冒充现场图。**
- **三档大小 = 重要度 × 想看清价值**：人脸 / 细节 / 想凑近看的→放大靠中心；通用图→缩小放边角。
- **⛔ 绝不隔日复用同一张图**：图片一眼就看出重复，是最露馅的偷懒。**每天现抓当天新闻的图，不准用昨天/近几天用过的任何一张图。** 同一条延续性新闻（重要、值得继续报）→ 换该新闻里的**另一张**图，或宁可不放（标"本条无配图"），**绝不重复同一张**。做法：维护一份 `used-images.log`，每天把用过的图源 URL 追加进去；选图前先查该 URL 最近几天没出现过再用。
- **对角线错位**：同模块相邻两条，图位一上一下，走左上→右下对角线，避免呆板。

## 6. 版式规范 —— 最高铁律，任何版本都必须遵守
> ⛔ **禁止所有新闻从上到下一条条堆叠（那是网页，不是报纸）。这是最容易犯、也最不可接受的错误。**

- **板块内 ≥2 条 → 必须左右互锁（dogleg）**：一条占"顶部通栏标题 + 左下正文"（L 形），另一条嵌右下角，中间发丝竖线分隔。或用"左文右图 / 左图右文"的并排块。**至少要有左右关系，不能全竖排。**
- **单条板块** → 占一宽条，正文双栏排。
- **有限块型**：hero（头条大块）/ 宽条 / 窄高 / 小块。按重要度分配，别每条独一份怪形状（守一致性，别让人看着累）。
- **分区要醒目**：板块用**色条 / 反色标题**（深底白字或红底白字），不是一根细线。面向中老年尤其要粗、要大、要一眼分得清。
- **响应式**：宽屏多列、平板 2 列、手机 1 列自动重排（照抄样板 CSS）。
- **眼动线**：标题永远在最上（图不许压在标题之上）；来源链接一律 `target="_blank"`。
- **字号随 profile**：中老年 / 退休版→正文 ≥18px、标题更大、行距更松；专业版→常规。

## 7. 报头
- **报名**（大字，短而有力）、**motto**（小字，可暖）、**报眉**（日期 + 星期 + 栏目词）——都读 `profile.md`。

## 8. 输出
- 一个完整 `.html`：`<style>` 抄对应样板，`<body>` 结构照它套，只换当天内容与图。文件名 `YYYY-MM-DD.html`。
- **拿不准就对着样板比**——样板是这套规范的标准答案。

## 9. 两个版式档案（受众举例）
| | 专业·AI 行业版 | 生活·本地版（中老年） |
|---|---|---|
| 样板 | `example-ai-*.html` | `example-local-*.html` |
| 板块 | 大模型·Agent / 出海·中日跨境 / 日本·企业落地 / 拓展·利于成长 | 头版(天气·安全) / 提防·街坊眼(八卦防骗) / 菜篮子·省钱 / 城里·城外(趣事) / 便民 |
| 腔调 | 通讯社简明体 | 晚报暖体 |
| 第三段 | 对你（机会） | 提醒您（贴心话） |
| 字号 | 常规 | 加大 |
| 难点 | 图片版权 | **超本地八卦 + 当天真图**（源头封闭，最难，见 §5） |
